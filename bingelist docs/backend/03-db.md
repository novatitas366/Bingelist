## What this file does

Opens (or creates) a SQLite database file and creates the three tables on first run. Exports:
- `db` — the database connection used by all route files
- `isUniqueViolation(e)` — helper to detect duplicate-key errors

---

## Key concepts

- **SQLite** — a file-based relational database. The entire database lives in `watchly.db`. No separate database server needed.
- **better-sqlite3** — synchronous SQLite driver. Unlike most Node database libraries, queries don't return Promises — they return results directly.
- **Prepared statements** — `db.prepare(sql)` compiles the SQL once. The returned statement is then called with `.run()`, `.get()`, or `.all()`. Parameters use `?` placeholders to prevent SQL injection.
- **WAL mode** — Write-Ahead Logging. Multiple readers can work simultaneously with a single writer without blocking each other.
- **Foreign keys** — disabled by default in SQLite; the pragma enables them so `ON DELETE CASCADE` works.

## Tables

### `users`
| Column | Type | Notes |
|---|---|---|
| `id` | INTEGER PK | auto-increments |
| `username` | TEXT UNIQUE | must be unique, 3–32 chars |
| `password_hash` | TEXT | bcrypt hash — never the plain password |
| `created_at` | TEXT | `datetime('now')` default |

### `watchlist`
| Column | Type | Notes |
|---|---|---|
| `id` | INTEGER PK | |
| `user_id` | INTEGER FK → users | `ON DELETE CASCADE` |
| `show_id` | INTEGER | TVMaze show ID |
| `show_name` | TEXT | cached so we don't always re-fetch |
| `show_image` | TEXT | nullable |
| `status` | TEXT | one of the 4 allowed values (CHECK constraint) |
| `note` | TEXT | nullable, user's personal note |
| `added_at` | TEXT | `datetime('now')` default |
| UNIQUE | `(user_id, show_id)` | one entry per show per user |

### `watched_episodes`
| Column | Type | Notes |
|---|---|---|
| `id` | INTEGER PK | |
| `user_id` | INTEGER FK → users | `ON DELETE CASCADE` |
| `show_id` | INTEGER | TVMaze show ID (for fast per-show queries) |
| `episode_id` | INTEGER | TVMaze episode ID |
| UNIQUE | `(user_id, episode_id)` | can't mark same episode twice |
